# 是什么
抽象类是用 `abstract` 修饰的类  
接口是用 `interface` 定义的类
# 作用
抽象类用于提取子类公共代码  
接口用于定义规范/契约, 规定能做什么
# 具体区别
抽象类不能实例化, 可以同时拥有抽象方法和具体方法  
接口不能实例化, 只能有抽象方法
# 应用场景
抽象类, 抽取子类共有的代码, 避免重复写  
比如 base 类, 作为基类提供公共能力  
接口定义你必须有什么能力, 不关心你怎么实现  
比如完全不相关的类有相同行为 —— 如 `Bird` 和 `Airplane` 都实现 `Flyable`
