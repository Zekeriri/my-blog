# 是什么
记忆化搜索就是递归 + 缓存  
# 作用
每个问题只算一次, 可以降低时间复杂度
# 原理
在暴力递归的基础上, 用一个备忘录记录已经算过的子问题结果, 遇到重复计算时直接返回缓存值  
和动态规划很像, 像是递归版的动态规划, 中间的状态转移方程都一样
# 什么时候用记忆化搜索?

## 适合用记忆化搜索

| 场景 | 例子 |
| ----------------- | ---------- |
| 状态转移关系复杂，难以确定循环顺序 | 树形 DP、区间 DP |
| 只需要计算部分状态（剪枝） | 博弈问题、搜索优化 |
| 状态是"集合/子集"（非连续索引） | 旅行商问题（TSP） |
| 保持递归思维更自然 | 面试手撕、算法原型 |

## 不适合用记忆化搜索

| 场景 | 原因 | 替代方案 |
|---|---|---|
| 状态空间连续且密集（1..n） | 递归开销大，且所有状态都要算 | 迭代 DP |
| n 很大（如 n=10^6） | 递归深度过大 → 栈溢出 | 迭代 DP |
| 追求极致性能 | 函数调用开销明显 | 迭代 DP + 滚动数组 |
| 需要空间优化 | 记忆化很难压缩到 O(1) | 迭代 DP 优化空间 |

# 语法

Python 官方实现的记忆化装饰器  
@cache 底层就是用 map 把结果存起来了, 下次用到时把结果给取出来

```python
from functools import cache
@cache
def fib(n):
    if n <= 1: return n
    return fib(n-1) + fib(n-2)
```

坑 1：参数必须可哈希（hashable）  
坑 2：类方法中的 self 导致缓存失效
