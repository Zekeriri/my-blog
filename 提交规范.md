# Git 提交说明规范

## 格式

```
类型：简述
```

一句话说清「做了什么」，不用写版本号。

## 类型

| 类型 | 什么时候用 |
|------|------------|
| 新增 | 新文章 |
| 修改 | 改已有文章 |
| 删除 | 删文章 |
| 配置 | 改分类、网站样式、部署等 |
| 修复 | 修 bug（页面报错、部署失败等） |
| 文档 | 改操作指南、README 等 |

## 示例

```text
新增：我的第一篇博客
新增：爬楼梯题解
修改：Redis 为什么快
配置：添加「想说的话」分类
修复：GitHub Pages 部署 404
文档：更新操作指南
```

## 注意

- 用中文冒号 `：`
- 简述写标题或改动点，10 字以内最好
- 一次 commit 只做一类事（例如：只发文章，或只改配置）
