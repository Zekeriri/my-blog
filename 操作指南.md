# 博客操作指南

## 本地预览

```powershell
npm install          # 首次或换电脑时
npm run dev          # 启动
```

浏览器打开 **http://localhost:4321/my-blog/**，停止按 `Ctrl + C`。

---

## Obsidian 写作

**仓库路径**（必须）：`c:\code\my-blog\src\content\blog`

### 一次性配置 Templater

1. 安装社区插件 **Templater** 并启用
2. **设置 → Templater**：
   - Template folder：`_templates`
   - 开启 **Trigger Templater on new file creation**
   - Folder Template：左侧 `/`，右侧 `_templates/文章模板.md`
3. **设置 → 文件与链接 → 新建笔记位置**：仓库根目录

### 日常写作

1. `Ctrl + N` 新建笔记，**文件名就是文章标题**（如 `我的第一篇博客`）
2. 模板自动填入日期等，`category` 需自己选
3. `Ctrl + Shift + F` 搜索全文
4. `Ctrl + S` 保存

## 文章格式

```md
---
pubDate: 2026-07-11
category: 算法题
tags: [动态规划, 数组]
---

正文……
```

**标题 = 文件名**（不用写 `title` 字段）。改文件名，网站标题会跟着变。

| 字段 | 必填 | 说明 |
|------|------|------|
| pubDate | 是 | 首次发布日期 |
| category | 是 | 大类 |
| tags | 否 | 标签 |

**更新时间自动获取**，不用手写。构建时读取该文件**最后一次 Git 提交时间**；若与 `pubDate` 同一天则不显示，否则网页显示「更新 xxx」。

### 换行

编辑器里按一次 Enter，网页上**不会换行**（会合成一段）。

| 想要的效果 | 写法 |
|------------|------|
| 同一段内换行 | 行末加**两个空格**，再按 Enter |
| 分成两段 | 中间**空一行**（按两次 Enter） |

示例（`过程:` 后面有两个空格）：

```md
一般递归的求解过程:  
把递归当成一个黑盒, 相信这个函数一定能解决问题,
```

### 写完后整理（提示词）

文章写好后，可复制下面这段给 AI，只做格式整理：

```text
帮我整理这篇 Markdown，正文意思和代码块都不动，只做三点：
1. 我手动换行的地方，行末加两个空格（让网页上和 Obsidian 里显示一致，单独成行）
2. 中文和英文/数字之间补一个空格（如：变成了 n=2、时间 O(n)）
3. 正文里出现的代码关键词用 `` 包起来（如：`abstract`、`try-catch`、`dp[i]`）
```

## 添加分类

改 `categories.json` 即可。重命名分类时，记得把已有文章 frontmatter 里的 `category:` 也改成新名字。

## 发布

```powershell
git add .
git commit -m "新增：文章标题"
git push
```

提交说明格式见 **[提交规范.md](./提交规范.md)**（如 `新增：爬楼梯题解`）。

首次部署前，必须在 GitHub 开启 Pages：

1. 打开 **Settings → Pages**
2. **Source** 选 **GitHub Actions**（不是 Deploy from a branch）
3. 保存后重新 push，或在 Actions 页手动 **Run workflow**
