# 博客操作指南

## 本地预览

```powershell
npm install          # 首次或换电脑时
npm run dev          # 启动
```

浏览器打开 **http://localhost:4321/my-blog/**，停止按 `Ctrl + C`。

---

## Obsidian 写作

**仓库路径**（必须）：`c:\code\my-blog\src\content\blog`

### 一次性配置 Templater

1. 安装社区插件 **Templater** 并启用
2. **设置 → Templater**：
   - Template folder：`_templates`
   - 开启 **Trigger Templater on new file creation**
   - Folder Template：左侧 `/`，右侧 `_templates/文章模板.md`
3. **设置 → 文件与链接 → 新建笔记位置**：仓库根目录

### 日常写作

1. `Ctrl + N` 新建笔记，**文件名用英文**（如 `binary-search`）
2. 模板自动填入，`title` 会用文件名，**`category` 需自己选**
3. `Ctrl + Shift + F` 搜索全文
4. `Ctrl + S` 保存

## 文章格式

```md
---
title: 标题
description: 摘要（可选）
pubDate: 2026-07-11
category: 算法题
tags: [动态规划, 数组]
---

正文……
```

| 字段 | 必填 |
|------|------|
| title | 是 |
| pubDate | 是 |
| category | 是 |
| description、tags | 否 |

---

## 添加分类

1. 改 `categories.json`（Obsidian 仓库根目录可见）
2. 在 `_meta/` 复制一份 `参考-*.md`，改成新分类名（用于 Obsidian 下拉选项）

---

## 发布

```powershell
git add .
git commit -m "新增：文章标题"
git push
```

提交说明格式见 **[提交规范.md](./提交规范.md)**（如 `新增：爬楼梯题解`）。

首次部署前，必须在 GitHub 开启 Pages：

1. 打开 **Settings → Pages**
2. **Source** 选 **GitHub Actions**（不是 Deploy from a branch）
3. 保存后重新 push，或在 Actions 页手动 **Run workflow**
